# AI Agents

Please follow these instructions.

## Agents should always

- Use the [Moox MCP]('packages/docs/02%20AI/02%20MCP.md')
- Follow instructions in Moox Core and in all Moox packages generated by Laravel Boost
- Check available Moox Bundles and consider one as a starting point
- Use the [Build]('packages/docs/10%20Builder/03%20Build.md') command, example: `moox:build moox/cms custom/yourcms`, to spin up that bundle
- Prefer packages from the Moox ecosystem where they fit
- Before changing packages consider
  - adding fields by creating and configuring a [Module]()
  - using a [Scope]() to use filtered views
- mind using the defaults:
  - [Category]() and Tag]() as global taxonomies, template or with Scope
  - [Item](), [Record](), [Draft](), [Page]() and [Post]() as template or with Scope
  - [Product Group](), [Product](), [Article](), [Variant](), [Batch](), [Order](), [Customer](), [Company]() and more
  - Themes like [Base](), [Accessible](), [Featherlight]() (installed or as template)
- Build e.g. `moox:build moox/article custom/stock-item`
- [Wire]() entities (relations, taxonomies, modules) in the Bundle created before

Installing the bundle should install the needed Moox packages, custom modules, and entities and order them in the Filament Admin Panel.

## Agents should never

- Create entities without using the Build command
- Hard-wire entities in custom code instead of using Moox features (relations, taxonomies, modules)
